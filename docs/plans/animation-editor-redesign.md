## Целевая модель данных: LAYER STACK (объединяет «параметры на тип» + «микс >2»)
Сейчас cfg: `{kinds:[k1,k2?], beats, amp, ov, color, cycle, outline, width, bg, bgrx, bgfill}`
— три ОБЩИХ ползунка на все слои, максимум 2 слоя.

Новая cfg добавляет опциональный `layers` (массив), сохраняя обратную совместимость:
```
cfg.layers = [
  { kind:"spin",   amp:30, ov:10, dir:+1, phase:0.0 },
  { kind:"bounce", amp:14, ov:8,  dir:+1, phase:0.5 },
  ...N
]
```
- Каждый слой — СВОЙ тип + СВОИ параметры (амплитуда/overshoot/направление/фазовый сдвиг).
- `dir` (+1/-1): разворот вращения (`r`→ -r) или вертикали движения (`y`→ 512-y), применяется
  ГЕНЕРИЧЕСКИ как пост-обработка ks рига — без переписывания 46 case в `buildProps`.
- `phase` (0..1): сдвиг времени keyframes слоя для разнообразия в миксе.
- `el_*` поэлементные слои тоже в стеке (применяются к иконке, не как риги — как сейчас).

### Обратная совместимость (ОБЯЗАТЕЛЬНО)
`makeAnimX` и save/load обязаны принимать СТАРУЮ форму (`kinds[]`+global amp/ov) И новую
(`layers[]`). Если `cfg.layers` нет — собрать его из `kinds`/`amp`/`ov` на лету. Сохранённые
паки (`packs/*/pack.json` поле `studio`) НЕ должны сломаться. Дефолты в `defaultCfg`/`randomCfg`
могут эмитить новую форму, но рендер читает обе.

## UI: панель «Анимация / Стиль» (`studio/index.html`, правый `#panel`)
Заменить два дропдауна `#kind`/`#kind2` + общие `#amp`/`#ov` на СТЕК СЛОЁВ:
- Список слоёв: каждая строка = [тип ▾] [amp] [ov] [⟲ dir] [фаза] [✕ удалить].
- Кнопка «＋ слой» добавляет слой (N без жёсткого лимита; разумный max ~4 для размера ≤64КБ).
- `#beats` (скорость/длина лупа) остаётся ГЛОБАЛЬНЫМ (одна длина лупа на эмодзи).
- Цвет/радуга/обводка/фон-плитка — без изменений.
- Превью обновляется на каждое изменение слоя (как сейчас `upd()`).

Реюз существующих токенов Lumina (`--accent-bright`, `.mini`, `.btn.sec`, ползунки) —
НИКАКОГО нового визуального языка, только расширение текущего.

## UI: агент-промтер (в блоке «Создать из промта»)
Добавить кнопку рядом с textarea `#prompt`, напр. «✨ Авто-промт»:
- onclick → `POST /api/prompt {idea: promptEl.value}` → подставить ответ в `#prompt`.
- Состояния: загрузка (~2-6с), ошибка (503 → «нет ключа», 400 → «введи идею»), успех.
- Не блокировать ручной ввод; пользователь правит результат перед «Генерировать».
- Реюз `.mini`/`.btn.sec` и `#genstatus` для статуса.

## UI: бренд-пресеты (новый дропдаун над стеком слоёв)
Готовые «сюжеты» из `docs/animation-playbook.md` → выставляют layer-stack + цвет одним кликом:
- db-пульс: `[{kind:"beat",amp:16}]` + скан (или `heartbeat`)
- ON-кольцо: `[{kind:"ring",amp:24}]`
- monitor-загрузка: `[{kind:"drawon"}]` или `el_assemble`
- new-курсор: `[{kind:"blink"}]`
- shield-удар: `[{kind:"elastic",amp:22},{kind:"ring",amp:14,phase:0.3}]`
- update-вращение: `[{kind:"spin",amp:0,dir:+1}]`
Пресеты — просто данные (объект `PRESETS_BRAND` в studio.js), легко расширять.

## Verification gates (агент обязан выполнить)
1. `node /tmp/verify_anim.cjs` (или эквивалент): для одиночных движений (bounce/drop/rise/hop/shake
   при amp=60) и миксов из 2-4 слоёв — ВСЕ s-значения трансформа конечные числа, 0 невалидных.
2. Старая форма cfg (`{kinds:["bounce"],amp:14,ov:10}`) и новая (`{layers:[...]}`) обе рендерятся
   без ошибок и без чёрного экрана.
3. `node --check studio/studio.js` — синтаксис ок.
4. Поднять студию, проверить вживую: дефолтные эмодзи (db=bounce) не чёрные; микс 3 слоёв виден;
   авто-промт заполняет поле; бренд-пресет применяется.
5. Десктоп + мобайл раскладка (`@media` уже есть) не разъезжается.
